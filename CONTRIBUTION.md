# 🤝 Contributing Guide (OJ 코딩 스터디)

이 문서는 이 저장소에 기여하기 위한 **기본 규칙 요약본**입니다.  
자세한 사용 방법은 `USAGE.md`를 참고하세요.

---

## 📌 기본 원칙

- 이 저장소는 **OJ(백준 / 프로그래머스) 스터디 전용**입니다.
- 모든 작업은 **GitHub 웹 UI**에서 진행합니다.
- **문제 하나 = Issue 하나 = PR 하나**를 원칙으로 합니다.

---

## 🧩 작업 순서 요약

1. **Problem Issue 생성**
2. **브랜치 생성**
3. **문제 폴더에 코드 업로드**
4. **Pull Request 생성**
5. **리뷰 후 머지**

---

## 📂 디렉터리 규칙 (중요)

### 문제 폴더 규칙
- 허용 형식:
  - `boj-숫자` (예: `boj-1000`)
  - `pg-숫자` (예: `pg-42576`)

### 사용자 폴더 규칙
- 반드시 **본인 GitHub ID 이름**으로 생성
- 다른 사람 폴더 수정 ❌

### 올바른 예시

- problems/boj-1000/yoon-somang/solution.cpp


---

## 🔀 Pull Request 규칙

PR 본문에는 반드시 다음을 포함해야 합니다.

```md
Issue: Closes #<문제 Issue 번호>
```

## **Weekly 자동화를 원할 경우 (선택)**  

- Weekly: #<Weekly Issue 번호>

## **🚫 허용되지 않는 작업**  

- /problems 외 디렉터리 수정
- 문제 Issue 없이 PR 생성
- 폴더 이름 규칙 위반
- 다른 사람 폴더 수정
- main(master)에 직접 커밋

위 항목 위반 시 PR은 자동으로 실패합니다.  

## **🏷 언어 라벨**  

- PR에 포함된 코드 확장자를 기준으로 자동 부착됩니다.
- 예: lang:Python, lang:C++, lang:Java

## **✅ 권장 사항**  

- 풀이 설명이 길 경우 README.md 추가
- 커밋 메시지는 간단하고 명확하게 작성
- 예: Solve boj-1000
