# 🛠 Maintainer Guide (운영자용)

이 문서는 스터디 운영자/리뷰어를 위한 가이드입니다.

---

## 👀 PR 리뷰 체크리스트

PR을 머지하기 전에 아래를 확인하세요.

- [ ] PR 상태가 모두 ✅ 통과인지
- [ ] `/problems/<문제>/<작성자>/` 구조가 맞는지
- [ ] 문제 Issue가 `Closes #번호`로 연결되어 있는지
- [ ] 코드가 최소한의 실행/논리 오류는 없는지

⚠️ 자동화가 통과되지 않은 PR은 **머지하지 않습니다.**

---

## 🔀 PR 머지 방법

1. PR 페이지 진입
2. Checks 전부 통과 확인
3. (필요 시) Approve
4. **Merge pull request** 클릭

---

## 🤖 머지 후 자동 처리 확인

머지 후 다음이 자동으로 수행됩니다.

- ✅ 문제 Issue에 커밋 목록 댓글
- ✅ Weekly Issue 체크박스 자동 `[x]`
- ✅ Weekly 달성률 댓글
- ✅ PR에 언어 라벨 부착

❌ 자동화가 안 되었다면:
- Actions 탭에서 워크플로우 로그 확인

---

## 🧹 운영 팁

### Weekly Issue 운영
- 주 1회 `[WEEKLY] YYYY-WW` Issue 생성
- 모든 참여자가 체크박스를 작성하도록 유도

### 문제 폴더 관리
- 동일 문제를 여러 명이 풀어도 OK
- 폴더 충돌은 GitHub ID 기준으로 방지됨

---

## 🚨 예외 처리 기준

- 규칙 위반 PR → Close + 이유 코멘트
- 초보자 실수 → 1회 설명 후 수정 유도
- 반복 위반 → 운영자 논의 후 조치

---

## 📌 권장 운영 철학

> 이 레포의 목적은  
> **“정답 공유”가 아니라 “과정 기록 + 피드백 문화”**입니다.

자동화는 규칙을 강제하기 위한 도구일 뿐,  
서로의 학습 경험을 존중하는 것이 최우선입니다.
