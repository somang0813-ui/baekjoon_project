
# 📄 `FAQ.md`

# ❓ FAQ – 자주 묻는 질문

이 문서는 PR 실패, 자동화 미작동 등  
스터디 중 자주 발생하는 문제에 대한 안내입니다.

## Q1. PR이 빨간 ❌ 로 실패했어요

### 가장 흔한 원인
- `/problems` 밖 파일을 수정함
- 문제 폴더 이름이 규칙에 맞지 않음  
  (`boj1000`, `boj_1000` ❌)
- 내 GitHub ID가 아닌 폴더를 수정함

👉 **PR → Checks 탭**에서 실패 사유를 확인하세요.

## Q2. Issue에 커밋 목록이 안 달렸어요

### 원인
- PR 본문에 `Closes #이슈번호`가 없음
- 이슈 번호 오타

### 해결
PR 본문에 아래 형식이 있는지 확인:
```md
Closes #12
```

## **Q3. Weekly Issue 체크가 자동으로 안 됐어요**
**확인할 것 3가지**
1. PR 본문에 Weekly: #번호가 있는가?
2. Weekly Issue에 체크박스가 있는가?
3. 체크박스의 문제 이름이 폴더명과 정확히 같은가?

**올바른 예**
- [ ] boj-1000

## **Q4. 언어 라벨이 안 붙었어요**
**원인**

- 확장자가 인식되지 않는 파일만 업로드
- (예: .txt, .md만 있는 경우)

**해결**

- 코드 파일 확장자 확인
- (.py, .cpp, .java 등)

## **Q5. PR 하나에 문제를 여러 개 올려도 되나요?**

**❌ 권장하지 않습니다.**
이 저장소의 원칙은:

문제 1개 = Issue 1개 = PR 1개

자동화 정확도를 위해 꼭 지켜주세요.  

## **Q6. 실수로 잘못 올렸어요. 어떻게 하나요?**

**PR 상태라면:**

파일 수정 후 다시 커밋 → PR 업데이트

**이미 머지된 경우:**

운영자에게 Issue 또는 댓글로 문의
